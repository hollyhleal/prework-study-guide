HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is a semantic HTML element. The name of a semantic HTML element describes its content and role it plays on the site.

CSS

  • Three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, external CSS style sheet
  • Inline CSS: apply to a specific HTML tag; useful for quick and permanent changes; less flexible than external and internal style sheets
  • Internal CSS Style Sheet: contains CSS rules for the webpage in the head section of the HTML file; rules only apply to that page
  • External CSS Style Sheet: most common and preferred; link the fil using a link element in the HTML's head
  • Margin property: indicates how much space we want around the outside of an element; not visible on the page; transparent buffer outside the element
  • Padding property: adds space around the content inside an element; increasing the padding makes the entire element bigger
  • CSS Box Model: Margin, Border, Padding, Content
  • Class attribute: allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • Head element: contains meta information for the browser; add links to related files here to tell browser what's included in our website and how these files should interact

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: make sure to run prior to committing to ensure your local branch is up-to-date with main branch

JavaScript

  • Script element should be added at the very bottom of the file, just before the closing body tag. The browser reads code in the order it appears in the file
  • Variables: containers that store values
  • JavaScript is case sensitive
  • Functions: ways of packaging functionality that you wish to reuse
  • Event Handlers: code structures that listen for activity in the browser, and run code in response
  • All primitive types, except null, can be tested by the typeof operator. typeof null returns "object"
  • Undefined indicates the absence of a value, while null indicates the absence of an object
  • Boolean: true or false
  • JavaScript strings are immutable; once a string is created, it is not possible to modify it
  • Array: single variable used to hold a group of data; typically used to hold data that is related in some way; a single array can store different types of data
  • Typed Arrays: present an array-like view of an underlying binary data buffer; offer many methods that have similar semantics to the array counterparts
  • JSON (JavaScript Object Notation): lightweight data-interchange format, derived from JavaScript, used by many programming languages
  • Control flow: the order in which a computer executes code in a script; in JS, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • Strict Equality Operator (===): checks to see if two values are equal and returns a Boolean result true if the values are equal and false if the values are not equal
  • If statements: allow us to interrupt control flow
  • For Loops: Keyword 'for'; three-part conditional statement; code block
